## v1.0.13 - 2026-04-07

## v1.0.12 - 2026-04-07

### 主要更新

- 修复 QQ 换取令牌默认短链路仍使用旧表单参数的问题，默认分支现在会优先复用官方 `authorize_url` 中的 `scope`、`redirect_uri`、`state` 等参数，不再因为 `scope=` 空值而被 QQ 反复 302 打回 `oauth2.0/show`
- 默认短链路的授权提交头和表单改为对齐 QQ 当前官方脚本，包括 `Referer: graph.qq.com/oauth2.0/show`、`Origin: https://graph.qq.com`、`from_ptlogin=1`、毫秒级 `auth_time` 与动态 `ui`
- 保留完整 `oauth2.0/show` 链路作为非默认备用，但后续实测主线改为这条“短链路 + 官方参数对齐”的实现
- 补充默认短链路和 `show` 失败后回退的回归测试，并在本地验证 `python tests/test_regressions.py` 共 67 项通过

## v1.0.11 - 2026-04-07

### 主要更新

- 默认屏蔽近几版持续失败的 `oauth2.0/show` 授权页链路，QQ 绑定改回优先使用更短的 legacy `authorize -> code -> qcCodeToOpenId` 兑换流程，只聚焦获取插件实际需要的 `openid/access_token`
- 保留 `oauth2.0/show` 授权页链路代码用于后续对照排查，但默认不再启用；仅测试或显式开关时才会进入该路径
- 当显式启用授权页链路且其再次被 QQ 打回 `oauth2.0/show` 时，仍会自动回退到 legacy 直连兑换链路，避免整次绑定直接失败
- 补充“默认 legacy 流程”和“show 链路失败后回退”的回归覆盖，并验证当前完整测试集共 66 项通过

## v1.0.10 - 2026-04-01

### 主要更新

- QQ 换取令牌阶段改为优先复用扫码前官方 `oauth2.0/show` 授权页的原始上下文，不再在扫码成功后临时重建一张新的授权页，修复授权请求被 QQ 重新打回展示页、始终拿不到授权码的问题
- 取码阶段会额外记录官方授权页 URL 与 `Q.isNeedLogin` 状态，并在后续授权提交时沿用对应 `Referer`、`update_auth` 和页面参数，更贴近官网真实浏览器流程
- QQ 授权提交请求补充官方同源表单场景下的 `Origin: https://graph.qq.com`，降低仅凭参数对齐但上下文仍被判定为无效的概率
- 补充“复用初始授权页上下文”的回归测试，并验证当前完整测试集共 65 项通过

## v1.0.9 - 2026-04-01

### 主要更新

- QQ 换取令牌阶段改为先回访官方 `authorize/show` 页面，再沿用页面中的 `redirect_uri`、`scope`、`state`、`src` 和 `Referer` 发起授权提交，进一步对齐官网当前授权流程
- 授权提交字段改为匹配官方脚本使用的 `from_ptlogin=1`、毫秒级 `auth_time`、动态 `ui` 和 `openapi=1010`，修复扫码成功后被重定向回授权展示页、始终拿不到授权码的问题
- 兼容 QQ 授权接口返回 JSON `callback` 的成功形式，不再只依赖 `302 Location`，并修复非 JSON 响应被误判成授权失败的分支问题
- 补充换令牌阶段对官方 `authorize/show` 页面和授权回调形式的回归测试，并验证当前完整测试集共 64 项通过

## v1.0.7 - 2026-04-01

### 主要更新

- QQ 换取令牌阶段改为沿用扫码阶段解析出的官方 `loginConfig`，保持 `Referer` 和登录上下文一致
- `oauth2.0/authorize` 后的授权跳转改为继续跟随官方白名单域名内的中转跳转，再从最终回跳地址提取授权码，修复“QQ 登录成功但换取令牌失败”的一类误判
- 当授权链路最终没有返回 `code` 时，提示文案从笼统的“Cookie过期，请重新扫码登录”改为更准确的“未获取到授权码，请重新扫码登录”

## v1.0.6 - 2026-04-01

### 主要更新

- QQ 扫码绑定链路改为动态解析腾讯当前 `xlogin` 页面配置，并将 `appid`、`ptui_version`、`target`、`style`、`href` 等参数贯穿到二维码生成与登录状态轮询，避免旧硬编码参数导致持续“获取登录状态失败”
- 放行腾讯当前实际使用的 `ssl.ptlogin2.graph.qq.com` 登录回跳域名，修复扫码确认后被插件误拦截的问题
- 文本接口请求改为先读取原始字节再按多编码兜底解码，降低上游返回非 UTF-8 内容时触发 `UnicodeDecodeError` 并导致 `df检查` 失败的概率
- 补充 QQ 登录配置提取、合法回跳域名放行与响应解码回退的回归测试

## v1.0.5 - 2026-04-01

### 主要更新

- 通知路由改为优先遵循 AstrBot 官方会话语义，基于 `unified_msg_origin`、`MessageType` 和 `event.is_private_chat()` 识别私聊/群聊，并兼容旧版 `friend:/group:` 目标数据
- 绑定失效、凭证异常等敏感提醒继续只发用户私聊；连续上游异常和物品目录自动刷新失败等系统告警改为优先发送到 AstrBot 全局配置 `admins_id` 对应的管理员私聊
- 新增官方私聊目标推导逻辑，即使用户最近一次交互发生在群聊，也会优先回推同平台私聊 session，避免敏感提醒误发到群
- 物品目录缓存自动刷新周期调整为 12 小时；若远程刷新失败，会沿用旧缓存继续检测并主动发送一次管理员告警
- 清理旧路由相关跳过测试，并补充管理员私聊告警、官方 session 路由和绑定提醒兜底的回归测试

## v1.0.4 - 2026-04-01

### 主要更新

- 登录重定向改为逐跳手动校验白名单，不再让 `aiohttp` 自动跟随后续未校验的 30x 跳转，并收窄 `REQUEST_EXCEPTIONS`，避免将本地 `ValueError` 误当成网络错误吞掉
- 运行数据目录在异常回退时改为标准 `data/plugin_data/astrbot_plugin_deltaforce_loot_broadcast/`，并继续兼容迁移旧版 `.runtime_data/`
- 凭证解密失败现在会显式标记为 `_secret_errors`，前台命令与后台轮询都会给出明确反馈，不再静默当成空值
- 同一发送者重新绑定账号时会重置旧账号的基线、待重试播报和角色信息，避免跨账号串状态
- 检查流程调整为先持久化最新基线再执行播报，降低超时或写盘失败导致重复播报的风险
- `df刷新物品缓存` 现在可以区分“远程刷新成功”和“仅回退到本地缓存”，不再误报刷新成功
- `df检查` 与 `df检查详细` 会更早识别凭证异常，群发送失败提示也不再回显原始异常字符串
- 补充回归测试，覆盖临时注入 `sys.modules`、重绑定状态清理、缓存回退提示和基线持久化顺序等场景

## v1.0.3 - 2026-03-31

### 主要更新

- 为 `GameAPI` 会话改用隔离的 `DummyCookieJar`，避免多用户共享轮询时出现 Cookie 串号和跨账号污染

- 清理 `red_detector.py` 中残留的损坏字符串与日志文案，统一调试摘要中的“撤离带出 / 带入局内”分类
- 补充对应回归测试，覆盖 CookieJar 选择、失败群重试和解绑/取消群绑定写盘失败场景
