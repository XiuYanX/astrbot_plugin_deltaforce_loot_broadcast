# 更新日志

## v1.0.3 - 2026-03-31

### 主要更新

- 为 `GameAPI` 会话改用隔离的 `DummyCookieJar`，避免多用户共享轮询时出现 Cookie 串号和跨账号污染
- 修复部分播报群发送失败后基线仍然前移的问题，失败目标会进入待重试队列，不再静默丢失这次播报
- 为 `df解绑`、`df取消群绑定` 和手动 `df检查` 的失败落盘场景补齐错误处理与提示文案
- 清理 `red_detector.py` 中残留的损坏字符串与日志文案，统一调试摘要中的“撤离带出 / 带入局内”分类
- 补充对应回归测试，覆盖 CookieJar 选择、失败群重试和解绑/取消群绑定写盘失败场景

## v1.0.2 - 2026-03-31

### 主要更新

- 复用统一的 `GameAPI` 实例，避免插件内部重复维护 `aiohttp.ClientSession`
- 增加登录重定向白名单校验，降低异常跳转带来的安全风险
- 修复 Cookie 合并逻辑，兼容 JSON、双重编码 JSON 和标准 Cookie Header 字符串
- 凭证保护失败时拒绝明文落盘，并为绑定/设群失败补充更明确的提示信息
- 统一播报群 `origin` 的归一化处理，修复重复设群反馈不准确的问题
- 优化最近对局补全流程，并提升批量用户检查时的异常容错能力
- 补充回归测试，覆盖存储失败、重复设群、Cookie 解析和批量检查异常场景

## v1.0.1 - 2026-03-30

### 首次发布

- 支持 QQ 扫码绑定三角洲行动账号
- 支持设置播报群、取消群绑定和查看当前状态
- 支持本地缓存物品目录，并按 6 级收集品进行播报
- 支持手动检查最近一局和导出调试报告
